Java基础之线程池（二）

在第一篇Java基础之线程池（一）介绍过线程成相关概念，最近在工作中又遇到一些相关线程池的知识点，所以有了这篇文章。

事情是这样的：在项目中使用spring的线程池，在任务执行的过程中出现了异常，但是在日志中心却找不到任何抛出的异常。因为这个异步方法是void类型的，因此猜想跟spring的线程池任务提交的方式有关。debug了一下源码，确实发现是这个原因。这个在接下来详细分析，我们先从线程池的异常处理开始。

线程池出现异常-异常处理

问题提出及回答

试问，如果一个线程池中的线程出现异常，那么线程池会如何处理这个异常？

我们首先回忆一下上一篇文章，如果是一个实现Callable接口的方式创建线程任务，在执行call方法的时候，如果抛出异常，那么会执行setException(ex)方法，将异常信息设置到outcome属性中。当我们调用get()方法的时候，则会获取到异常信息。

其次，我们知道线程池的提交方式有两种：

  1. execute
  2. submit

这两种提交方式有什么区别，这也是经常考的一个问题。

首先，execute是接口Executor的方法，接受一个Runnable参数，返回值为void；

而submit方法是接口ExecutorService的方法，接受参数可以为Callable参数或者Runnable参数和result作为返回值。submit方法返回的是一个Future结果。关于Future接口，我们上篇也提到过，它有好几个方法，其中get()方法则可以获取到 任务执行结果。

其次，是就是我们这一小节要讲的，对异常的处理方式不同。例如下面例子，我们用两种不同方式提交任务，观察结果：

可以看出：

当执行方式是execute时，是有堆栈异常信息打印的；而执行方式是submit时，没有堆栈异常输出。

但是，当我们使用submit方法返回的Future结果调用get()后，可以看出有异常信息打印，如下图。这是因为执行submit方法时，将这这个Runnable包装为FutureTask，这跟我们上面回忆的第一点相吻合。

所以，回到我们这小节的开篇问题上：如果一个线程池中的线程出现异常，那么线程池会如何处理这个问题？

正确回答应该是：

这与线程的提交方式有关，如果使用的execute方式提交，那么会抛出异常。如果是submit方式提交，是将任务包装成Callable，用FutureTask进行包装，将返回结果或者异常信息封装在Future中，当get的时候，则会抛出异常信息。

了解了线程池的异常处理原理，那么我们可以怎么做能够感知到这些异常信息呢？

  1. 还记得runWorker里面的finally的protected void afterExecute(Runnable r, Throwable t) { }，这是一个空实现，我们可以在这里对抛出的异常进行处理，设置提醒之类的。
  2. 设置线程池的UncaughtExceptionHandler。这样当JVM发现未被捕获的异常后会调用UncaughtExceptionHandler里面的uncaughtException方法处理异常。当然，以上两种方式不适用Callable方式的线程任务。
  3. 如果使用submit方法执行任务，那么利用Future对象的get方法抛出的异常来进行处理。

线程池出现异常-线程处理

问题提出及回答

试问，如果一个线程池中的线程出现异常，那么线程池会如何处理这个线程？

我们上面解决完如果讨论完当线程池中的线程出现异常，怎么处理这个异常。接下来我们讨论，那既然这个线程都出现了异常，那么线程池会怎么处理这个异常，是会把它放会线程池吗？还是进行其他处理？

答案是：线程会移除这个线程，重新new 一个worker来继续处理任务。

源码解析

所以，综上，Spring的线程池将任务包装为Callable形式，以submit方式提交的。所以当方法为void时，线程池将异常信息通过setException()方法设置起来，没有抛出，又因为方法返回值为void，所以就没有异常打印。

@Async的异常处理

所以对于@Async标注的方法：

  1. 如果方法为void，那么可以通过配置AsyncUncaughtExceptionHandler来捕获异常。在执行invoke方法时，执行invocation.proceed()会捕获这个方法抛出的异常，执行AsyncUncaughtExceptionHandler的handleUncaughtException方法，因此，我们可以在这里来对异常进行处理。

  1. 如果方法有返回值，那么返回AsyncResult类的对象，因为AsyncResult是Future的子类，直接通过future.get()即可捕获异常。
